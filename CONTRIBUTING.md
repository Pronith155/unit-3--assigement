# Contributing Guidelines

## Welcome
Thank you for choosing to contribute to this project!  
These simple rules ensure the project stays clean and organised.

---

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button at the top-right to create your own copy.

### 2. Create a Branch
Use a clear name like:
- feature/add-new-tool
- fix/bug-in-script
- docs/update-readme

### 3. Make Your Changes
Edit files, fix issues, or add new features.

### 4. Write Clear Commit Messages
Examples:
- Add new function for discount calculation
- Update README with installation steps
- Fix typo in script

### 5. Open a Pull Request
Go to **Pull Requests â†’ New Pull Request**  
Describe what you changed and why.

---

## Code Style
- Keep code simple and readable  
- Add comments for tricky logic  
- Follow the folder structure already used  

---

## Documentation
If your change affects usage, update the README or docs.

---

## Code of Conduct
Be respectful and follow the project rules.
